**Current code (lines 194-201):**
```bash
# Find last numbered phase before backlog (999.x)
LAST_PHASE=$(ls -d .planning/phases/[0-9]*/ 2>/dev/null | grep -v "999" | sort -V | tail -1 | grep -oE '[0-9]+' | head -1)

if [ -n "$LAST_PHASE" ]; then
  NEXT_PHASE=$((LAST_PHASE + 1))
  echo "Non-blocking issues automatically routed to phase ${NEXT_PHASE} (pre-backlog)."
fi
```

**Replace with:**
```bash
# Read phase prefixes from config (DEBT, PARK, WANT)
CLEANUP_PREFIX=$(node -pe "require('./.planning/config.json').phase_conventions?.cleanup_prefix || 'DEBT'")
BACKLOG_PREFIX=$(node -pe "require('./.planning/config.json').phase_conventions?.backlog_prefix || 'PARK'")

# Find last numbered phase before special phases (DEBT.x, PARK.x, WANT.x)
# Sequential phases are numeric only; filter out alphabetic prefixes
LAST_PHASE=$(ls -d .planning/phases/[0-9]*/ 2>/dev/null | sort -V | tail -1 | grep -oE '^[0-9]+' | head -1)

if [ -n "$LAST_PHASE" ]; then
  NEXT_PHASE=$((LAST_PHASE + 1))
  echo "Non-blocking issues automatically routed to phase ${NEXT_PHASE}."
fi
```

**Key changes:**
1. Add config reads for CLEANUP_PREFIX and BACKLOG_PREFIX at the start of the step
2. Change filter logic: instead of `grep -v "999"`, use `[0-9]*/` pattern which naturally excludes DEBT.x, PARK.x, WANT.x (alphabetic prefixes)
3. Remove "pre-backlog" terminology since we now use semantic names

**Important:** The filtering change is simpler — sequential phases start with digits (01, 02, 36.1), while special phases start with letters (DEBT.1, PARK.1, WANT.1). The glob `[0-9]*/` already excludes alphabetic prefixes, so we just need to read the config for informational purposes (and to support any future use of these variables in the skill).
