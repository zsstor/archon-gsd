### CR-01: Command Injection in `_validate_escalation_chain`

**File:** `/home/zzs/dev/.meta/bin/lib/escalation.sh:424`
**Issue:** Python heredoc uses environment variable `CHAIN_JSON` but the heredoc delimiter is not quoted. This allows shell expansion of backticks, `$()`, and other shell metacharacters within the heredoc body before Python sees it.

**Current code:**
```bash
CHAIN_JSON="$chain_json" python3 <<EOF 2>/dev/null
import json
import sys
import os

try:
    chain = json.loads(os.environ['CHAIN_JSON'])
# ...
EOF
```

**Fix:** Quote the heredoc delimiter to prevent shell expansion:
```bash
CHAIN_JSON="$chain_json" python3 <<'EOF' 2>/dev/null
import json
import sys
import os

try:
    chain = json.loads(os.environ['CHAIN_JSON'])
# ...
EOF
```

### CR-02: Command Injection in `query_history`

**File:** `/home/zzs/dev/.meta/bin/lib/logging.sh:131`
**Issue:** Python heredoc uses direct shell variable interpolation (`$max_age_days`, `$task_type`, `'$DELEGATION_LOG'`) instead of environment variables. Variables are substituted by the shell before Python sees them, enabling command injection if variables contain shell metacharacters.

**Current code:**
```bash
python3 <<EOF 2>/dev/null || echo "[]"
import json
from datetime import datetime, timedelta

results = []
cutoff = datetime.now() - timedelta(days=$max_age_days)

try:
    with open('$DELEGATION_LOG') as f:
        for line in f:
            try:
                entry = json.loads(line.strip())
                if entry.get('task_type') != '$task_type':
                    continue
# ...
EOF
```

**Fix:** Use environment variables and quote the heredoc delimiter:
```bash
MAX_AGE_DAYS="$max_age_days" TASK_TYPE="$task_type" DELEGATION_LOG="$DELEGATION_LOG" python3 <<'EOF' 2>/dev/null || echo "[]"
import json
import os
from datetime import datetime, timedelta

results = []
max_age_days = int(os.environ['MAX_AGE_DAYS'])
cutoff = datetime.now() - timedelta(days=max_age_days)

task_type = os.environ['TASK_TYPE']
delegation_log = os.environ['DELEGATION_LOG']

try:
    with open(delegation_log) as f:
        for line in f:
            try:
                entry = json.loads(line.strip())
                if entry.get('task_type') != task_type:
                    continue
# ...
EOF
```

### CR-05: Command Injection in `read_config`

**File:** `/home/zzs/dev/.meta/bin/ai-delegate:107`
**Issue:** Python fallback for config reading uses shell variable interpolation.

**Current code:**
```bash
value=$(python3 -c "
import json
try:
    with open('$CONFIG_FILE') as f:
        cfg = json.load(f)
    # Navigate nested keys (e.g., 'task_routing.impl')

```

**Fix:** Use command-line arguments or environment variables:
```bash

import json
import os
import sys

try:
    config_file = os.environ['CONFIG_FILE']
    key = os.environ['KEY']

    with open(config_file) as f:
        cfg = json.load(f)
    # Navigate nested keys (e.g., 'task_routing.impl')
    keys = key.split('.')
    v = cfg
    for k in keys:
        if isinstance(v, dict):
            v = v.get(k)
        else:
            v = None
            break
    if v is not None:
        if isinstance(v, bool):
            print('true' if v else 'false')
        elif isinstance(v, (list, dict)):
            print(json.dumps(v))
        else:
            print(v)
except:
    pass
```

## Warnings

### WR-01: File Descriptor Collision Risk

**File:** `/home/zzs/dev/.meta/bin/lib/escalation.sh:200-204`
**Issue:** Hard-coded file descriptor 200 for flock could collide with other operations in complex scripts. While unlikely in practice, this violates defensive programming principles.

**Current code:**
```bash
[[ -s "$temp_file" ]] && {
    flock -x 200 || true
    cat "$temp_file" >> "$history_file"
    rm -f "$temp_file"
} 200>>"$history_file"
```

**Fix:** Use a higher file descriptor number (e.g., 9) or dynamically allocate:
```bash
[[ -s "$temp_file" ]] && {
    flock -x 9 || true
    cat "$temp_file" >> "$history_file"
    rm -f "$temp_file"
} 9>>"$history_file"
```

Or better yet, use `flock` with a file path directly:
```bash
[[ -s "$temp_file" ]] && {
    flock "$history_file" bash -c "cat '$temp_file' >> '$history_file'"
    rm -f "$temp_file"
}
```

### WR-02: Incomplete Credential Sanitization Pattern

**File:** `/home/zzs/dev/.meta/bin/lib/logging.sh:266`
**Issue:** The regex pattern `[^[:space:]&]*` may not catch credentials followed by semicolons, quotes, or other delimiters common in URLs or JSON.

**Current code:**
```bash
sanitized=$(echo "$sanitized" | sed -E 's/(password|key|token|secret|api_key)\s*=\s*[^[:space:]&]*/\1=REDACTED/gi')
```

**Fix:** Expand the character class to include more delimiters:
```bash
sanitized=$(echo "$sanitized" | sed -E 's/(password|key|token|secret|api_key)\s*=\s*[^[:space:]&;"\x27]*/\1=REDACTED/gi')
```

Or more robustly, match until known delimiters:
```bash
sanitized=$(echo "$sanitized" | sed -E 's/(password|key|token|secret|api_key)\s*=\s*([^[:space:]&;",]+)/\1=REDACTED/gi')
```

## Info

### IN-01: Inconsistent Security Pattern Application

**Files:** All reviewed files
**Issue:** The codebase now has three different patterns for Python heredocs:

1. ✅ **Correct:** Env vars + quoted heredoc (e.g., `escalation.sh:41-67`)
2. ❌ **Vulnerable:** Env vars + unquoted heredoc (if any existed)
3. ❌ **Vulnerable:** Direct shell interpolation in heredoc (e.g., `logging.sh:131`)

This inconsistency makes it easy for future developers to accidentally introduce vulnerabilities by copy-pasting the wrong pattern.

**Fix:** Establish a single secure pattern and apply it consistently:

**Secure Pattern Template:**
```bash
# Pass data via environment variables, use quoted heredoc
VAR1="$value1" VAR2="$value2" python3 <<'EOF'
import os
var1 = os.environ['VAR1']
var2 = os.environ['VAR2']
# ... Python code using var1, var2 ...
EOF
```

**Action items:**
1. Update all Python heredocs to use this pattern
2. Document in `CLAUDE.md` or `CONTRIBUTING.md`
3. Add shellcheck rule or pre-commit hook to detect unquoted heredocs with variables
