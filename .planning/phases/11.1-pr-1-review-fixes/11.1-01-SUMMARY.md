## Technical Details

### Environment Variable Pattern (CR-01, WR-04)

**Before (vulnerable):**
```bash
python3 <<EOF
chain = json.loads('''$chain_json''')
EOF
```

**After (safe):**
```bash
CHAIN_JSON="$chain_json" python3 <<'EOF'
import os
chain = json.loads(os.environ['CHAIN_JSON'])
EOF
```

### Flock Atomic Append Pattern (WR-02)

**Before (race-prone):**
```bash
cat "$temp_file" >> "$history_file" && rm -f "$temp_file"
```

**After (atomic):**
```bash
{
    flock -x 200 || true
    cat "$temp_file" >> "$history_file"
    rm -f "$temp_file"
} 200>>"$history_file"
```

### Python3 Availability Check Pattern (WR-03)

**Before (silent failure):**
```bash
python3 <<'EOF' 2>/dev/null || echo "60"
# ...
EOF
```

**After (explicit check):**
```bash
if ! command -v python3 &>/dev/null; then
    echo "60"
    return
fi

result=$(python3 <<'EOF'
# ...
EOF
) || result=""

[[ "$result" =~ ^[0-9]+$ ]] && echo "$result" || echo "60"
```

### MAX_PARALLEL Semaphore Pattern (WR-05)

**Before (unbounded):**
```bash
while read task; do
    ( ... ) &
    pids+=($!)
done
```

**After (bounded):**
```bash
MAX_PARALLEL="${MAX_PARALLEL:-4}"
running=0

while read task; do
    while [[ $running -ge $MAX_PARALLEL ]]; do
        wait -n 2>/dev/null || true
        ((running--)) || true
    done

    ( ... ) &
    pids+=($!)
    ((running++)) || true
done
```
