Create `~/dev/.meta/bin/lib/routing.sh` with these functions per D-01, D-02, D-03, D-04, D-10:

**Public functions (exported):**

1. `route_task()` - Main entry point
   - Args: task_type, description, context_file (optional)
   - Returns: selected model name
   - Calls: `_score_complexity`, `_compute_weighted_failure_rate`, `_check_upgrade_trigger`, `_select_model`
   - Per D-04: Internal score is 0-100 numeric

2. `_get_scoring_mode()` - Get current mode for task type
   - Args: task_type
   - Returns: "keyword" | "keyword+files" | "full"
   - Per D-02: Default to "keyword" if no config entry exists
   - Read from `.planning/config.json` under `task_routing.<type>.scoring_mode`
   - If key missing, return "keyword"

3. `_score_complexity()` - Compute complexity score
   - Args: task_type, description
   - Returns: integer 0-100
   - Per D-01: Three modes:
     - `keyword`: Scan description only. Baseline 50. High complexity keywords (refactor, migrate, architect, multi-file) add 35. Medium keywords (test, integrate, api, database) add 15. Cap at 100.
     - `keyword+files`: keyword score + file count bonus. If description mentions >3 files, add 20. If >6 files, add 40.
     - `full`: keyword+files + context analysis. If context_file provided and >200 lines, add 25.

4. `_check_upgrade_trigger()` - Check if scoring mode should upgrade
   - Args: task_type, failure_rate, escalation_avg
   - Returns: 0 (should upgrade) or 1 (no upgrade)
   - Per D-03: Dual-signal trigger:
     - failure_rate > 0.3 (30%) triggers upgrade, OR
     - escalation_avg > 2.0 triggers upgrade
   - Log upgrade decision if triggered

5. `_select_model()` - Select best model from chain
   - Args: task_type, complexity_score
   - Returns: model name
   - Read task_routing section from config.json
   - Apply thresholds: score <40 = first model, 40-70 = second model, >70 = third model
   - Check availability via `model-registry check $model`
   - Return first available model that matches threshold

**Internal helpers:**

- `_score_keywords()` - Keyword-only scoring (0-100)
- `_count_affected_files()` - Parse file count from description

**Header format:**

```bash
# routing.sh - Complexity scoring, history lookup, model selection
# Sourced by ai-delegate, not executed directly
# Implements: D-01, D-02, D-03, D-04, D-10

set -euo pipefail

# Dependencies: expects SCRIPT_DIR, CONFIG_FILE, DELEGATION_LOG set by parent
```

   - Calls: `_run_backend`, `_handle_timeout`, `_capture_output`
