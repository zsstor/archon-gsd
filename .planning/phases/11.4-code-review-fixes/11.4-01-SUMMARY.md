## Security Impact

**CR-01: Command Injection (CRITICAL) - FIXED**
- **Before:** Task type parameter passed via string interpolation in jq selectors
- **After:** Task type passed via jq --arg flag, making injection impossible
- **Attack prevented:** `task_type="; rm -rf /; "` would have executed arbitrary commands
- **Test coverage:** 3 tests verify semicolon, quote, and command substitution attacks are blocked

**WR-02: Config Corruption (HIGH) - FIXED**
- **Before:** Direct write to config file with `open(file, 'w')`
- **After:** Atomic write via temp file + move operation
- **Impact:** SIGTERM during write no longer corrupts config
- **Pattern:** tempfile.mkstemp in same directory + shutil.move (atomic on POSIX)

**WR-01: Array Out-of-Bounds (MEDIUM) - FIXED**
- **Before:** `${models[2]}` returned empty string if array had <3 elements
- **After:** `${models[2]:-${models[0]:-$DEFAULT_MODEL}}` always returns valid model
- **Impact:** Model selection never fails silently

## Performance Impact

**WR-03: Float Comparison Speedup**
- **Before:** `python3 -c "exit(0 if $x > Y else 1)"` ~100ms startup overhead
- **After:** `awk -v x="$x" 'BEGIN { exit(x > Y ? 0 : 1) }'` <1ms execution
- **Improvement:** ~100x faster for simple float comparisons
- **Availability:** awk is POSIX standard, no new dependencies

## Architecture Notes

**Pattern: Nested Fallback for Array Access**
```bash
# Single fallback (old pattern)
${models[0]:-gemini-flash}

# Nested fallback (new pattern)
${models[2]:-${models[0]:-$DEFAULT_MODEL}}
```
Benefits:
- Falls back to first model if high-complexity model unavailable
- Falls back to constant if array is completely empty
- Three levels of defense against empty selection

**Pattern: Environment Variable Passing to Python**
```bash
# Old (vulnerable to injection)
python3 <<EOF
task_type = '$task_type'
EOF

# New (injection-safe)
TASK_TYPE="$task_type" python3 <<'EOF'
task_type = os.environ['TASK_TYPE']
EOF
```
Benefits:
- Single quotes on EOF prevent shell variable expansion
- Environment variables are not subject to quoting/escaping issues
- Works consistently regardless of task_type content
