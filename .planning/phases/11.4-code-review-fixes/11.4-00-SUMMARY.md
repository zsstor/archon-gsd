## Test Methodology

**Pattern:** Follow test_quota.sh structure
- Shebang and comment header with requirements
- set -uo pipefail (not -e for failure testing)
- Test counters (TESTS_RUN, TESTS_PASSED, TESTS_FAILED, TESTS_SKIPPED)
- Reusable assert functions (assert_exit_code, assert_output_contains, etc.)
- Isolated TEST_DIR with cleanup trap
- Structured test sections by requirement ID

**Security Testing Approach:**
- Command injection: Test with malicious payloads (semicolons, quotes, substitutions)
- Array bounds: Test with out-of-range indices
- File operations: Test with missing, unreadable files
- Profile sourcing: Test opt-in vs opt-out behavior with environment markers
- Size limits: Test with large files to verify truncation

**Mock Strategy:**
- Mock model-registry for routing tests
- Mock config files in TEST_DIR
- Mock delegation log with test data
- Mock profile with detectable markers
- Mock modules for dependency testing
