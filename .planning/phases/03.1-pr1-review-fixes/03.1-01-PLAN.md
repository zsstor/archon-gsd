**Problem:** ai-delegate uses `set -euo pipefail` at line 23. When escalation.sh is sourced, this setting is inherited. At line 269 of escalation.sh, `execute_model` is called and may return non-zero (e.g., 2 for quota, 124 for timeout, 127 for unavailable). With `set -e` active, any non-zero return causes immediate script termination BEFORE `exit_code=$?` can capture the return code.

**Fix:** Temporarily disable errexit around the execute_model call to allow proper exit code capture.

Find the execute_model call section in `execute_with_escalation()` (around lines 267-270):
```bash
        local start_time exit_code
        start_time=$(date +%s%3N)
        execute_model "$model" "$prompt" "$output_file" "$timeout"
        exit_code=$?
```

Replace with:
```bash
        local start_time exit_code
        start_time=$(date +%s%3N)
        # Temporarily disable errexit for model execution
        # ai-delegate uses set -e which breaks escalation loop on non-zero returns
        set +e
        execute_model "$model" "$prompt" "$output_file" "$timeout"
        exit_code=$?
        set -e
```

This ensures:
1. exit_code properly captures non-zero returns (2, 124, 127, etc.)
2. The escalation loop can continue to try the next model
3. errexit is re-enabled after capture so other failures are still caught
