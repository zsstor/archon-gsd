# Phase 11.5 Plan 01: Security and Correctness Hardening Summary

- **IN-01 (Important):** Fixed jq fallback chain to properly fall back to impl.models
- **WR-07 (Warning):** Added file locking to prevent config race conditions
- **WR-03 (Warning):** Fixed Codex backend to respect model selection parameter
- **IN-02 (Important):** Escaped markdown fences in handoff context to prevent prompt injection

## Tasks Completed

### Task 1: Fix Python heredoc injection in routing.sh (CR-01)

**Status:** ✓ Complete
**Commit:** .meta@4fb8188
**Files Modified:** ~/dev/.meta/bin/lib/routing.sh

**What was done:**
- Changed `python3 <<EOF` to `python3 <<'EOF'` (quoted EOF) in two functions
- Replaced inline shell variable interpolation with env var passing
- Applied to `_get_avg_escalation_count` and `_compute_weighted_failure_rate`

**Pattern used:**
```bash
avg=$(DELEGATION_LOG="$DELEGATION_LOG" TASK_TYPE="$task_type" python3 <<'EOF' 2>/dev/null
import os
delegation_log = os.environ['DELEGATION_LOG']
task_type = os.environ['TASK_TYPE']
# ... rest of script
EOF
)
```

**Security impact:** Eliminates command injection vector where malicious input in `$task_type` or `$DELEGATION_LOG` could execute arbitrary code.

### Task 2: Fix jq fallback chain and config race condition (IN-01, WR-07)

**Status:** ✓ Complete
**Commit:** .meta@647beaa
**Files Modified:** ~/dev/.meta/bin/lib/routing.sh

**What was done:**

1. **Fixed jq fallback (IN-01):**
   - Restructured jq expression to use parentheses for scoping
   - Changed from `... | ... // .task_routing.impl.models` to `(...) // (.task_routing.impl ...)`
   - Now properly falls back to impl.models when task type not found

2. **Added file locking (WR-07):**
   - Wrapped `_upgrade_scoring_mode` in flock subshell
   - Uses file descriptor 9 with `${CONFIG_FILE}.lock`
   - Prevents concurrent config modifications from racing

**Correctness impact:** Tasks with unknown types now correctly fall back to impl chain instead of failing. Concurrent config updates no longer risk corruption.

### Task 3: Fix Codex model selection and handoff escaping (WR-03, IN-02)

**Status:** ✓ Complete
**Commit:** .meta@f0d2991
**Files Modified:** ~/dev/.meta/bin/lib/execution.sh, ~/dev/.meta/bin/lib/escalation.sh

**What was done:**

1. **Fixed Codex model selection (WR-03):**
   - Added model parameter to `_run_codex_backend` function
   - Implemented model alias mapping (codex|o3 → o3, gpt-5.4 → gpt-5.4)
   - Updated dispatcher to pass model: `_run_codex_backend "$model" "$prompt" "$output_file"`
   - Changed from hardcoded `--model o3` to dynamic `--model "$codex_model"`

2. **Escaped handoff context (IN-02):**
   - Added sed pipe to escape triple backticks: `sed 's/```/` ` `/g'`
   - Prevents failed-model output from terminating markdown fence early
   - Protects escalated prompt from being reshaped by malicious/broken output

**Security impact:** Model selection now respects config instead of always using o3. Handoff context can no longer inject markdown that alters the next model's prompt structure.
