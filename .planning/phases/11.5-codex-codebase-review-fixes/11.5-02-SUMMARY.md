# Phase 11.5 Plan 02: AI Delegate Robustness Fixes Summary

**One-liner:** Fixed set -e exit code capture, final model attribution, JSONL locking with flock, and review/code-review task type consistency.

## Objectives Achieved

Fixed five critical reliability issues in ai-delegate identified by Codex codebase review:
- WR-01: set -e breaks failure paths - exit codes not captured
- WR-02: wrong model in outcome logs - initial model logged instead of final escalated model
- WR-04: JSONL without locking - race condition in concurrent writes
- WR-05: silent logging failures - errors discarded with || true
- WR-06: review vs code-review inconsistency - mismatch with quota.sh

## Tasks Completed

### Task 1: Fix set -e handling in ai-delegate command handlers
**Status:** Complete
**Commit:** b63dfca
**Files:** ~/dev/.meta/bin/ai-delegate

Fixed all four command handlers (cmd_impl, cmd_test_pass, cmd_scaffold, cmd_review) to wrap `execute_with_escalation` calls in if/else blocks. This prevents `set -e` from terminating the script before exit codes can be captured and logged.

**Pattern applied:**
```bash
# Before: exit_code=$? never runs if execute_with_escalation fails
execute_with_escalation "impl" "$model" "$prompt" "$output_file" "$AI_TIMEOUT"
exit_code=$?

# After: exit code always captured
if execute_with_escalation "impl" "$model" "$prompt" "$output_file" "$AI_TIMEOUT"; then
    exit_code=0
else
    exit_code=$?
fi
```

### Task 2: Fix outcome attribution to use final model
**Status:** Complete
**Commit:** f64a9fc
**Files:** ~/dev/.meta/bin/ai-delegate, ~/dev/.meta/bin/lib/escalation.sh

Updated escalation.sh to write final_model to file alongside escalation_count. All four ai-delegate handlers now read final_model and pass it to log_task_outcome instead of the initial routed model.

**Changes:**
- escalation.sh success path: `echo "$model" > "${output_file}.final_model"`
- escalation.sh failure path: `echo "${chain[$((${#chain[@]}-1))]}" > "${output_file}.final_model"`
- ai-delegate handlers: Read .final_model, use in log_task_outcome calls

### Task 3: Add file locking and failure warnings to logging.sh
**Status:** Complete
**Commit:** 3d1f4b1
**Files:** ~/dev/.meta/bin/lib/logging.sh

Added flock-based locking to all JSONL append operations and rotation functions. Replaced silent `|| true` failures with warning messages to stderr.

**Functions updated:**
- log_routing_decision: flock + warning
- log_task_outcome: flock + warning
- log_escalation_event: flock + warning
- log_quota_event: flock + warning
- _rotate_log: flock for atomic rotation
- _rotate_escalation_log: flock for atomic rotation

**Pattern:**
```bash
{
    flock -x 9 || { echo "[logging] Warning: Could not acquire log lock" >&2; }
    # ... python heredoc ...
} 9>>"$DELEGATION_LOG" || echo "[logging] Warning: Failed to write to $DELEGATION_LOG" >&2
```

### Task 4: Normalize task type 'review' to 'code-review'
**Status:** Complete
**Commit:** 255fa8b
**Files:** ~/dev/.meta/bin/ai-delegate

Changed cmd_review to use "code-review" task type throughout instead of "review". This matches quota.sh constraints and config expectations where code-review has cross-family-preferred routing.

**All occurrences updated:**
- Task type variable: `local task_type="code-review"`
- All routing calls: `route_task "$task_type" ...`
- Escalation calls: `execute_with_escalation "$task_type" ...`
- Logging calls: `log_task_outcome "$task_type" ...`

## Key Technical Decisions

1. **File descriptor 9 for flock:** Consistent pattern across all logging functions to avoid conflicts with stdin/stdout/stderr.

2. **Fallback to initial model:** `final_model=$(cat "${output_file}.final_model" 2>/dev/null || echo "$model")` ensures backwards compatibility if .final_model file is missing.

3. **Warning format:** All logging warnings follow `[logging] Warning: <detail>` pattern for grep-ability.

4. **Task type early binding:** Moved task_type variable to top of cmd_review for single-source-of-truth.
