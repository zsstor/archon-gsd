  <action>
Update `migration_status()` function in `.archon/lib/migration.sh` (lines 107-122) to check `is_partial_migration` BEFORE `is_gsd_project`.

Current (incorrect order):
```bash
migration_status() {
  local project_root="${1:-.}"

  if is_zarchon_migrated "${project_root}"; then
    echo "zarchon"
  elif is_gsd_project "${project_root}"; then
    echo "gsd"
  elif is_partial_migration "${project_root}"; then
    echo "partial"
  elif [ -d "${project_root}/.planning" ]; then
    echo "unknown"
  else
    echo "fresh"
  fi
}
```

Target (correct order):
```bash
migration_status() {
  local project_root="${1:-.}"

  if is_zarchon_migrated "${project_root}"; then
    echo "zarchon"
  elif is_partial_migration "${project_root}"; then
    echo "partial"
  elif is_gsd_project "${project_root}"; then
    echo "gsd"
  elif [ -d "${project_root}/.planning" ]; then
    echo "unknown"
  else
    echo "fresh"
  fi
}
```

The issue: `is_gsd_project()` returns true for any project that has `.planning/` and is NOT fully migrated. This includes partially-migrated projects (have `.archon/` but no `zarchon_version`). By checking `is_partial_migration` first, we correctly identify interrupted migrations as "partial" instead of "gsd".
  </action>
  <verify>
    <automated>grep -A 15 '^migration_status()' .archon/lib/migration.sh | grep -B 2 'is_gsd_project' | grep -q 'is_partial_migration' && echo "PASS: partial check comes before gsd check"</automated>
  </verify>
  <acceptance_criteria>

  </acceptance_criteria>
  <done>migration_status checks is_partial_migration before is_gsd_project</done>
