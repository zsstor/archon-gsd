## Technical Details

### Fix 1: task_routing.review Config

**Problem:** ai-delegate's review command called routing functions with "review" task type but config only had "code-review" entry, causing fallback to default "gemini-flash" model.

**Evidence:**
```bash
# ai-delegate:360-368
complexity_score=$(_score_complexity "review" "Code review: $target")
failure_rate=$(_compute_weighted_failure_rate "review")
model=$(route_task "review" "Code review: $target")
# ... but then:
model_chain=$(_get_model_chain "code-review")  # ← different key!
```

**Solution:** Added task_routing.review entry matching code-review's configuration.

### Fix 2: migration_status() Detection Order

**Problem:** is_gsd_project() returns true for projects with .planning/ that aren't fully migrated. This includes partial migrations (have .archon/ but no zarchon_version), so partial migrations were misclassified as "gsd".

**Logic flow before:**
```
if is_zarchon_migrated → "zarchon"
elif is_gsd_project → "gsd"  ← catches partials here!
elif is_partial_migration → "partial"  ← never reached for partials
```

**Logic flow after:**
```
if is_zarchon_migrated → "zarchon"
elif is_partial_migration → "partial"  ← catches partials here
elif is_gsd_project → "gsd"
```

**Verification:** Tested on current zarchon project, correctly returns "zarchon".
